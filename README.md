### Organisation du Projet

Le projet Nomina-v3 est organisé en :

- **EPICs** : Grands thèmes fonctionnels (ex: EPIC G - CONTENU & CRUD, EPIC H - Relations & navigation)
- **User Stories (US)** : Stories individuelles rattachées aux EPICs (ex: US-G1, US-G2)
- **Sprints** : Les issues contiennent des références aux sprints (S1, S2, etc.)
- **Priorités** : Chaque issue a une priorité (P0, P1, etc.)

## About the Project

Nomina-v3 est un système de gestion de contenu narratif permettant de gérer des lieux, catégories, cultures, personnages, concepts, titres et fragments d'histoire.

### Fonctionnalités Principales

Les EPICs actuels incluent :
- **EPIC G** : CRUD pour le modèle de données (Lieux, Catégories, Cultures, etc.)
- **EPIC H** : Relations & navigation entre entités
- **EPIC I** : Recherche, filtres et tri
- **EPIC J** : Import/Export/Seed
- **EPIC K** : Éditeurs & expérience d'écriture
- **EPIC L** : Auth, permissions & modération
- **EPIC M** : Tests, qualité & surveillance
- **EPIC N** : Génération et randomisation narrative
- **EPIC O** : Internationalisation & localisation
